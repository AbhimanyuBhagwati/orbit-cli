How It Works

Five-stage pipeline.

The LLM handles exactly 2 things: decomposing goals and generating plans. Everything else is deterministic code.

1

Environment Scan

Five async collectors run in parallel — git, Docker, Kubernetes, system, and filesystem. Fault-tolerant with 5s timeouts, cached with 5s TTL. Token-aware budget allocates context by relevance.

Deterministic
2

Task Decomposition

Goal broken into subtasks, each tagged with a capability: fast_shell code_gen reasoning

LLM Call
3

Model Selection

Each subtask routed to the best locally-available model. Walks a priority list of known-good models, checks what's installed via Ollama, returns the first match.

Deterministic
4

Plan Generation

Structured output via Pydantic JSON schema. Every plan step has a command, expected outcome, and risk classification. Validated before execution.

LLM Call
5

Execution Loop

For each step: classify risk (regex), confirm based on tier, execute via async subprocess, observe result. On failure: auto-replan — 15 steps max, 3 replans per step, 25 total LLM calls.

Why not use the LLM for safety?

Using the same system that generates commands to judge their safety is a circular dependency. It's vulnerable to prompt injection, can hallucinate, and takes seconds instead of microseconds. Regex patterns are deterministic, injection-immune, and instant.

Features

Everything you need.

Built for real DevOps workflows, not demos.

Runbooks

Save successful workflows as YAML runbooks. Replay with orbit runbook run. Shareable, version-controllable DevOps recipes.

Diagnostics

Run orbit wtf after a failure. Full context analysis with error classification and suggested fixes.

Rollback Plans

Destructive commands get automatic rollback plans. Git reflog recovery, kubectl delete reversal, compose restarts.

Command History

SQLite-backed searchable log of every action. Browse with orbit history. Powers the diagnostics engine.

Semantic Memory

Optional ChromaDB vector store for semantic search over past actions. Install with orbit-cli[rag].

Multi-Provider

Ollama by default. Optional OpenAI and Anthropic providers. Structured output validated against Pydantic schemas.
